Look in pipeline

A LinkedPipes ETL pipeline can be executed via the REST API. It is possible to pass files to the pipeline via an HTTP POST request and these files appear as output of the {{page.title}} component. The typical usage is passing different input files to a fixed pipeline via the API, e.g. in a larger, repetitive data processing workflow.

API call sample

The HTTP POST request can be executed e.g. like this: curl -i -X POST -H "Content-Type: multipart/form-data" -F "input=@data1.tmp" -F "input=@data2.tmp" http://demo.etl.linkedpipes.com/resources/executions?pipeline=http://demo.etl.linkedpipes.com/resources/pipelines/created-1464778953280 where data1.tmp and data2.tmp are the files which will appear in the output of the component.
